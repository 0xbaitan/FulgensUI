## 11. Notes & Decisions

### Design Decisions

**React Context for theme state**

- **Context:** Need to share theme state across app
- **Decision:** Use React Context with Provider/hook pattern
- **Rationale:** Standard React pattern, no external deps needed
- **Alternatives Considered:**
  - Zustand store (rejected: overkill for single value)
  - Props drilling (rejected: not scalable)
  - CSS variables only (rejected: need React state for Toggle)

**localStorage for persistence**

- **Context:** User preference should persist across sessions
- **Decision:** Store theme preference in localStorage with key `fulgensui-theme`
- **Rationale:** Simple, widely supported, no backend needed
- **Alternatives Considered:**
  - Cookies (rejected: unnecessary for client-only state)
  - IndexedDB (rejected: overkill)
  - URL query param (rejected: pollutes URLs)

**PandaCSS `_dark` selector**

- **Context:** Apply dark mode styles
- **Decision:** Use PandaCSS conditional styles (`_dark` pseudo)
- **Rationale:** Already configured in project
- **Alternatives Considered:**
  - CSS variables (rejected: not PandaCSS pattern)
  - Data attributes (rejected: less semantic)
