# Example Planning Document

This directory contains a reference example of a complete component planning document generated by the Designer Agent.

## Purpose

This example demonstrates:

- ✅ **Complete planning document structure** - All sections filled out with realistic content
- ✅ **Design token mappings** - How Penpot colors/spacing map to PandaCSS tokens
- ✅ **Validation reports** - Spacing grid, color compliance, accessibility, state coverage
- ✅ **PandaCSS recipe specification** - Complete recipe structure ready for implementation
- ✅ **Test specifications** - Vitest test requirements
- ✅ **Storybook specifications** - Story definitions and variants
- ✅ **Implementation checklist** - Step-by-step tasks for coder agents

## Usage

When the Designer Agent is invoked to create a planning document for a new component from Penpot, it should follow the same structure and level of detail as shown in `COMPONENT-PLAN.md`.

## File Structure

```
specs/examples/button/
├── README.md              # This file
├── COMPONENT-PLAN.md      # Complete planning document example
└── assets/                # Placeholder for exported assets
    ├── button-default.png      # Default state (would be exported from Penpot)
    ├── button-variants.png     # All variants (would be exported from Penpot)
    └── button-states.png       # Interactive states (would be exported from Penpot)
```

## Key Features Demonstrated

### 1. Design Token Mappings

Shows how to map Penpot hex colors to PandaCSS token references with status indicators:
- ✅ Existing tokens
- ⚠️ New tokens needed
- ❌ Invalid hard-coded values

### 2. Validation Reports

Demonstrates all five validation checks:
- Spacing grid adherence (4px/8px multiples)
- Color token compliance
- Accessibility contrast (WCAG AA/AAA)
- Interactive state coverage
- Responsive design considerations

### 3. Complete Recipe Specification

Provides exact PandaCSS recipe structure including:
- Base styles with measurements from Penpot
- All variants with token references
- Interactive states (_hover, _focus, _active, _disabled)
- Default variant configuration

### 4. Implementation Guidance

Shows how to provide clear guidance to coder agents:
- Exact file paths and naming conventions
- Step-by-step implementation checklist
- Test specifications with example code
- Storybook story structure
- CI validation commands

## Reference

When creating planning documents, use this example as a template for:
- Section structure and ordering
- Markdown formatting and tables
- Status indicator usage
- Token mapping notation
- Code block formatting
- Checklist structure
