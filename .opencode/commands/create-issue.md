# /create-issue Command

Create a GitHub issue from an enhanced markdown file or @manager spec.

## Arguments

- `<file>`: Required - Path to the markdown file (.md) or YAML template (.yaml)
- `--dry-run`: Optional - Preview what would be created without calling GitHub API

## Supported Formats

1. **Manager Specs** (`.md` from `.temp/issues/specs/`)
   - Generated by `@manager` agent
   - Contains frontmatter + GitHub metadata
   - Uses `scripts/issue/parse_spec.py` for parsing

2. **Legacy YAML Templates** (`.yaml` from `.github/ISSUE_TEMPLATE/`)
   - Legacy format (use @manager instead for new issues)
   - Uses existing `scripts/issue/parser.py` for parsing

## Workflow

### Step 1: Detect Format and Parse

**For Manager Specs** (`.md` files in `.temp/issues/specs/`):

```bash
# Parse spec to JSON
python3 scripts/issue/parse_spec.py <file> --output /tmp/issue_data.json
```

This outputs JSON with:

- `title`: Issue title
- `body`: Markdown body
- `labels`: Array of labels (e.g., `["status:backlog", "issue:bug", "priority:must-have"]`)
- `project_id`: GitHub project ID
- `project_fields`: Dict of field_id → value_id
- `blocked_by`: Issue number (optional)
- `parent_link`: Issue number (optional)

**For Legacy YAML** (`.yaml` files):

Use existing parser (see legacy workflow below).

### Step 2: Validate Required Fields

Check:

- `title` must exist and not be empty
- `labels` must contain at least one `issue:*` label

### Step 3: Dry Run Mode

If `--dry-run`:

```bash
# Read parsed JSON
cat /tmp/issue_data.json | python3 -c "
import sys, json
data = json.load(sys.stdin)

print('🔍 Dry Run: <file>')
print()
print('Issue Details:')
print(f'  Title: {data[\"title\"]}')
print(f'  Labels: {', '.join(data[\"labels\"])}')
if 'blocked_by' in data:
    print(f'  Blocked By: #{data[\"blocked_by\"]}')
if 'parent_link' in data:
    print(f'  Parent: #{data[\"parent_link\"]}')
print()
print('Body Preview:')
print('---')
print(data['body'][:500])  # First 500 chars
if len(data['body']) > 500:
    print('...')
print('---')
print()
print('Would create issue on GitHub (dry-run)')
"
```

Exit without creating issue.

### Step 4: Check GitHub Auth

Run:

```bash
gh auth status
```

If not authenticated, show error and exit.

### Step 5: Create Issue

```bash
# Read parsed data
ISSUE_DATA=$(cat /tmp/issue_data.json)
TITLE=$(echo "$ISSUE_DATA" | python3 -c "import sys, json; print(json.load(sys.stdin)['title'])")
BODY=$(echo "$ISSUE_DATA" | python3 -c "import sys, json; print(json.load(sys.stdin)['body'])")
LABELS=$(echo "$ISSUE_DATA" | python3 -c "import sys, json; print(','.join(json.load(sys.stdin)['labels']))")

# Create issue
ISSUE_URL=$(gh issue create \
  --title "$TITLE" \
  --body "$BODY" \
  --label "$LABELS" \
  --repo "0xbaitan/FulgensUI" \
  --json url --jq .url)

# Extract issue number
ISSUE_NUM=$(basename "$ISSUE_URL")

echo "ISSUE_URL=$ISSUE_URL"
echo "ISSUE_NUM=$ISSUE_NUM"
```

### Step 7: Handle Relationships

```bash
# Read relationships from parsed JSON
BLOCKED_BY=$(echo "$ISSUE_DATA" | python3 -c "import sys, json; d=json.load(sys.stdin); print(d.get('blocked_by', ''))")
PARENT_LINK=$(echo "$ISSUE_DATA" | python3 -c "import sys, json; d=json.load(sys.stdin); print(d.get('parent_link', ''))")

if [ -n "$BLOCKED_BY" ]; then
  gh issue comment "$ISSUE_NUM" --body "⚠️ This issue is blocked by #$BLOCKED_BY" --repo "0xbaitan/FulgensUI"
fi

if [ -n "$PARENT_LINK" ]; then
  gh issue comment "$ISSUE_NUM" --body "📌 Parent issue: #$PARENT_LINK" --repo "0xbaitan/FulgensUI"
fi
```
